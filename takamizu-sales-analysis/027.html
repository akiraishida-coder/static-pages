たかみず・書き起こし音声フォールバック 進捗と手順

この文書の位置づけ

セールス原本の書き起こしが「字幕では取れず、音声＋Whisper で埋める」ときの手順と、バッチ再実行のメモである。チーム内で「まだ未取得か／分析を発話ベースに直したか」を突き合わせるチェックリストとして使う。共有フォルダを見せるとき、先頭が日付だけのファイル名だと中身が分かりにくいため、ファイル名は用途が分かる日本語にしている（作成きっかけは 2026-03-31）。

目的

セールス原本に「書き起こし取得失敗」が残っている件について、字幕APIに依存せず セールス原本_YouTube書き起こし取り込み.py の --audio-fallback（yt-dlp 音声＋faster-whisper）で順次埋め、その後に各セールス分析をトーク根拠で確定版に差し替える。

一括順次実行（推奨）

実処理は たかみず_失敗原本_音声書き起こし順次.sh（ファイル一覧の編集もここ）。.command は同 .sh を bash で呼ぶ薄いラッパーです。

ターミナルから確実に動かすとき（推奨）:

cd /Users/tanakayuichi/Projects/theleadpromotion
bash ツール/書き起こし/たかみず_失敗原本_音声書き起こし順次.sh

Mac の Finder からは たかみず_失敗原本_音声書き起こし順次.command をダブルクリック。

運用の全体像・トラブルシュート: ツール/書き起こし/セールス原本YouTube書き起こし_運用仕様.md

ログインが必要な限定公開動画がある場合は、.sh 内の python3 行の末尾に次を足す。

--cookies-from-browser chrome

（または safari）

1本だけ再実行

cd /Users/tanakayuichi/Projects/theleadpromotion
python3 ツール/書き起こし/セールス原本_YouTube書き起こし取り込み.py \
